### What the prompt must cover

- **Voice cloning flow:** user signs up via Clerk → immediately prompted to record ~5 seconds of their voice → Voxtral clones it → user hears "WHOA, that's my voice" playback. The "WHOA moment" is the activation event for this funnel.
- **Team-builder wizard:** after voice clone, user picks 3 agent slots by role (Frontend, Backend, DevOps). Each slot can be:
  - Named (user-chosen name — their personality)
  - Voiced (cloned-from-user, OR picked from a voice library)
  - Modeled (which LLM tier the agent thinks at — fast/deep/high-effort per `feedback-models-are-thinking-not-brain.md`)
- **Activation:** at the end of the wizard, user sees the $39/mo Basic tier Stripe checkout. Stripe Elements mounts with `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` (Clara Code account `acct_1TLoccKN7SZDz8BX`). On successful subscription, the 3 slots go live and the user enters `claracode.ai/dashboard` with their team ready to greet them.
- **Persistence:** agent slots + voice clone metadata persist to Neon Postgres (schema work required — new tables or extend existing User table).
- **Clara as concierge, NOT builder:** Clara orchestrates the team-builder UX but doesn't write code herself. The user's 3 agents are the builders. See `decision-clara-doesnt-build-team-builds.md`.
