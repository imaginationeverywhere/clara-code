No último mês o autor ganhou muitos seguidores (YouTube, Instagram, TikTok) em grande parte por um **sistema de conteúdo com Claude Code**: **7 skills** que sustentaram ~**10 milhões de visualizações** em ~30 dias como **one-man team** (sem editor, sem VA). São **~90 peças** (30 longas ~1 vídeo/dia em março + 60 shorts). O número-chave não é só views nem followers — é **volume sustentável** (90 vídeos) sem depender de um único viral; muitos “jab”, nenhum “haymaker” gigante.

**Método:** mapear o processo de criação → quebrar em fases → **automatizar cada parte** com skills específicas. Quatro fases: **pesquisa**, **ideação**, **roteiro** (hook, outline, título, thumbnail text), **distribuição** (repurpose). Pesquisa e ideação são um **ciclo** (pesquisa gera ideias → ideias pedem mais pesquisa).

**Skill principal — “YouTube pipeline” + Notebook LM:** integra **Notebook LM via CLI** dentro do Claude Code — manda URLs/PDFs/etc.; processamento pesado roda nos **servidores Google** (economiza tokens do Claude); traz de volta áudios, slides, etc. O pipeline **orquestra** sourcing + análise em um comando. Pré-requisito: já ter **fonte** do que falar.

**“Fonte zero” (o que falar primeiro):** no nicho dele (tech/AI), muito nasce em **GitHub** e **Twitter/X** antes do YouTube. Ele fez o Claude Code construir **app de scraping Twitter** → **Telegram** a cada 30–45 min: keywords/autores, score (velocidade, autoridade, timing, oportunidade, “replyability”), **Supabase** para deduplicar/diversificar, **softmax** para não sempre pegar o #1, sugestões de resposta (ex. Grok), loop de melhoria das respostas. Segunda fonte: script diário de **GitHub trending** (espaço AI) → **Obsidian** (top 10 da semana, top 5 do mês, stars, linguagem, sugestão do dia).

Com ideia bruta, encaixa no **pipeline** → Notebook LM analisa; tudo versionado em **markdown no Obsidian** para o humano acompanhar (evita “sumir” no meio só de código).

**Ideação:** skill olha a pesquisa no **panorama competitivo** — ângulos saturados, lacunas, outliers de performance; propõe **títulos, ângulos, desejo do público, formatos**, rankeia opções. Mensagem central: Claude Code como **colaborador** — o humano **permanece no comando**; não deixar o sistema ir do zero ao roteiro final sem checkpoints ou o resultado vira genérico. Cada etapa: revisar saída antes da próxima.

**Roteiro:** ele **scripta forte o hook** (~30s) com skill dedicada; o resto é **outline/bullets** e tom mais “live”. Skills de **hook**, **outline**, **título** (inspiradas em conteúdo do Callaway) — hooks com variações (falado, visual, texto para short); outline com seções, possíveis visuais (Excalidraw), fontes; títulos em **tiers** usando **histórico de performance** dos próprios vídeos + opções de texto para thumbnail. Thumbnail **manual** (IA sozinha é fraca para isso). Filmagem humana (sem avatar IA).

**Distribuição:** postar manualmente em algumas redes (CapCut → YouTube/TikTok; Instagram rápido manual). Foco em **repurpose**: **Content cascade** — transcript do YouTube → **blog** (SEO, embed), **thread Twitter**, variações **LinkedIn** (com ferramentas tipo Lead Shark); evita “AI writing tropes”. **Short form skill** — condensa hooks/outlines para clips 30/60/90s. De um longo nascem várias superfícies (blog, X, LinkedIn, shorts, Reels, TikTok).

**Sistema = colaborador esteróide:** delega análise, pesquisa competitiva, hooks, ideação base — humano foca no alto nível e na voz. Skills/GitHub script/Twitter engine no **Chase AI Plus** (link comentários/descrição) + “Claude Code Masterclass”.
