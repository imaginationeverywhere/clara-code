`ReactCurrentOwner` which no longer exists in React 19. `react-reconciler@0.33.0`

directly targets and resolves the root cause.

## Issues

### Medium — `engines` field absent despite docs claiming otherwise

**Files:** `packages/cli/package.json`, `docs/cli-voice-loop.md`

Both the commit message and the new doc section state:
> "Ink 6 keeps `engines: node >= 20`"

However, `packages/cli/package.json` has `"engines": {}` (empty object). Ink 6
itself only declares `engines: { node: ">=12.0.0" }` — it does not enforce our
Node 20 floor. A user on Node 18 will install `@clara/cli` via npx with no
warning, then fail at runtime on features that require Node 20 (e.g., the native
`node:test` runner, fetch API, crypto.subtle).

**Fix:** Add `"engines": { "node": ">=20.0.0" }` to `packages/cli/package.json`.
This causes npm/npx to emit a warning on incompatible Node versions before the
crash, and accurately reflects what the code actually requires.

**Priority:** Medium (not a blocker — boot crash is fixed, this is a UX degradation
for a narrow user segment. Ship now, fix in PR #5 preamble.)

---

### Low — `react: "^19.2.0"` minor floor bump

**File:** `packages/cli/package.json`

Bumping from `^19.0.0` to `^19.2.0` raises the minimum React minor version. In
an npm workspace where another package declares `react: "^19.0.0"`, npm will
satisfy both with 19.2.x — no conflict in practice. But it's worth noting the
floor has moved. Not a bug, but future contributors should know that React 19.0
or 19.1 can no longer satisfy this package's direct dep declaration.

**Priority:** Low / informational. No action required.
