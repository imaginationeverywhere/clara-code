### P2 — `isConnecting` guard exceeds spec

**File:** `ide/clara-code/extensions/clara-voice/src/extension.ts`

The prompt specified `isConnecting = false` in `"open"` and `"error"` callbacks. The Cursor agent
correctly added it to the `"close"` callback as well. This handles the edge case where a WebSocket
closes before `"open"` fires (e.g., immediate server rejection returning a `"close"` event without
`"error"`). Without this, `isConnecting` would be stuck at `true`, permanently locking the toggle.
This is a better implementation than the spec.

### P3 — `PropsWithChildren` is idiomatically correct

**File:** `packages/cli/src/tui.tsx`

Using `PropsWithChildren<{...}>` is the standard React pattern for components that accept children.
It's more maintainable than manually adding `children?: ReactNode` because it stays aligned with
React's type definitions across versions.

### P4 — Single shebang fix in CLI dist

**File:** `packages/cli/tsup.config.ts`

The `banner: { js: "#!/usr/bin/env node" }` was removed from the tsup config. The shebang already
exists in `src/index.ts`. With two shebangs, Node was choking on the second one as invalid JS on
the second line. The Cursor agent identified and removed the root cause rather than papering over
the symptom.

### P5 — JSX syntax in `commands/tui.tsx`

Switching from `React.createElement(App, {...})` to `<App .../>` is idiomatic for a `.tsx` file and
avoids the TypeScript inference issue that caused the original error. Correct call.
